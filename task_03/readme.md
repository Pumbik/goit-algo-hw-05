## Висновки

### Стаття 1 (Алгоритми)
Найшвидшим алгоритмом для цього тексту виявився **Boyer-Moore**.
- **Пошук існуючого підрядка:** Boyer-Moore (0.00113 с) більш ніж у 2 рази швидший за KMP і майже в 6 разів швидший за Rabin-Karp.
- **Пошук неіснуючого підрядка:** Перевага Boyer-Moore ще відчутніша. Він упорався за 0.05 с, тоді як KMP знадобилося 0.17 с, а Rabin-Karp — 0.32 с.

### Стаття 2 (Бази даних)
Найшвидшим алгоритмом для цього тексту також виявився **Boyer-Moore**.
- Тенденція збереглася: Boyer-Moore показав найкращий час як для знаходження слова (0.00067 с), так і для повного проходу тексту при пошуку відсутнього слова (0.079 с).
- Rabin-Karp показав найгірший результат, особливо при повному проході тексту (майже 0.5 с).

### Загальний висновок

На основі отриманих емпіричних даних можна зробити наступні висновки щодо ефективності алгоритмів:

1.  **Boyer-Moore — беззаперечний лідер.**
    Цей алгоритм показав найкращі результати у всіх тестах. Його ефективність пояснюється механізмом зсувів ("поганого символу"), що дозволяє пропускати частини тексту, не перевіряючи кожен символ. Чим довший підрядок і чим більший алфавіт, тим ефективніше він працює.

2.  **Kнута-Морріса-Пратта (KMP) — стабільне "срібло".**
    KMP працює швидше за Рабіна-Карпа, але повільніше за Боєра-Мура. Його головна перевага — лінійна складність $O(N)$, він не робить зайвих проходів по тексту, але перевіряє більше символів, ніж Боєр-Мур.

3.  **Rabin-Karp — найповільніший.**
    Алгоритм виявився найменш ефективним для задачі пошуку одного підрядка. Це зумовлено накладними витратами на постійний перерахунок хешу та обробку колізій. Цей алгоритм доцільніше використовувати для специфічних задач, наприклад, пошуку *множини* підрядків одночасно (плагіат), де він може перевершити інші методи.

**Підсумок:** Для задач пошуку підрядка у звичайних текстових даних алгоритм **Боєра-Мура** є найбільш оптимальним вибором за критерієм швидкодії.
